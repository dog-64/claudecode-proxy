# AGENTS.md — Руководство для агентов

Этот файл содержит инструкции для agentic coding agents, работающих в репозитории claudecode-proxy.

## Описание проекта

**claudecode-proxy** — набор bash-скриптов для запуска Claude Code через SSH-туннель и squid-прокси с поддержкой моделей GLM (Z.ai) и Anthropic (Sonnet/Opus/Haiku).

### Ключевые файлы

| Файл | Назначение |
|------|------------|
| `clc` | Основной скрипт запуска Claude Code |
| `install.sh` | Скрипт установки в bin-каталог |
| `Makefile` | Команды сборки и проверки |

---

## Команды сборки, проверки и установки

### Проверка скриптов (lint)

```bash
make shellcheck
```

Или напрямую:
```bash
shellcheck --severity=warning clc
shellcheck install.sh
```

**ВАЖНО:** Запускать `shellcheck` после **каждого** изменения `.sh` файлов или bash-скриптов. Исправлять все ошибки и предупреждения перед коммитом.

### Установка

```bash
make install     # Установка в /usr/local/bin/clc
./install.sh     # Интерактивная установка в выбранный bin-каталог
```

### Тестирование

В проекте нет автоматических тестов. Проверка выполняется вручную:

```bash
# Проверка GLM (без прокси)
./clc --model glm

# Проверка Sonnet (через прокси)
./clc --model sonnet
```

## Стиль кода

### Bash-скрипты

#### Форматирование

- Отступ: 4 пробела (без табов)
- Максимальная длина строки: 100 символов
- Shebang: `#!/bin/bash`
- Использовать `set -e` для выхода при ошибках

#### Именование

- Переменные: `UPPER_CASE` для констант, `lower_case` для локальных
- Функции: `lower_case_with_underscores`
- Файлы: строчные буквы без расширения для исполняемых скриптов

#### Структура скрипта

```bash
#!/bin/bash

set -e

# Краткое описание скрипта
# Использование: script [аргументы]

# Константы
CONSTANT_VALUE="value"

# Функции
function_name() {
    local var="value"
    # ...
}

# Основная логика
main() {
    # ...
}

main "$@"
```

#### Обработка ошибок

- Проверять обязательные переменные окружения:
  ```bash
  [[ -z "$REQUIRED_VAR" ]] && { echo "❌ REQUIRED_VAR не установлен" >&2; exit 1; }
  ```
- Использовать `>&2` для вывода в stderr
- Возвращать ненулевой код при ошибках: `exit 1`

#### Условия и циклы

```bash
# Предпочитаемый стиль if
if [[ -z "$VAR" ]]; then
    echo "Empty"
fi

# Цикл while
while [[ $# -gt 0 ]]; do
    case "$1" in
        --option)
            VALUE="$2"
            shift 2
            ;;
        *)
            break
            ;;
    esac
done
```

#### Подстановка команд


## Архитектура

### Модели и прокси

| Модель | API | Прокси | Переменные |
|--------|-----|--------|------------|
| `glm` | Z.ai | Нет | `ZAI_API_TOKEN` |
| `sonnet/opus/haiku` | Anthropic | Да (SSH-туннель) | Стандартные Anthropic |

### Функции в clc

- `setup_zai()` — настройка окружения для GLM через Z.ai
- `setup_anthropic()` — запуск SSH-туннеля и настройка прокси

### Переменные окружения

```bash
# Для GLM (Z.ai)
ZAI_API_TOKEN="токен"

# Для Anthropic (автоматически через прокси)
http_proxy="http://127.0.0.1:1080"
https_proxy="http://127.0.0.1:1080"
```
